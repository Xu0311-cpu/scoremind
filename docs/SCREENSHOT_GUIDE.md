# Screenshot Guide

Use these screenshots for GitHub, portfolio, and presentation materials. Prefer the sample file `frontend/public/samples/c_major_progression.musicxml` for the main flow and `frontend/public/samples/carried_context_notes.musicxml` for note-level context examples.

## Score Input Workspace

Capture the full Score Input Workspace showing all six source options with their status labels.

Suggested caption:

> Score Input Workspace shows supported, export-first, research-only, and out-of-scope score sources with distinct visual labels.

## Upload & Score Preview

Capture the upload panel and rendered MusicXML score preview.

Suggested caption:

> ScoreMind accepts MusicXML/XML files and renders a score preview for visual verification before deterministic analysis.

## Student Analysis

Capture Student Summary after running analysis, including global key, chord progression, and harmonic function explanations.

Suggested caption:

> Student Analysis summarizes global key, detected chords, harmonic functions, and note-level counts in a polished Chinese reading flow.

## Measure Walkthrough

Capture measure-level walkthrough cards showing chord labels, Roman numerals, and note summaries.

Suggested caption:

> Measure Walkthrough connects each measure's chord labels, Roman numerals, harmonic functions, and note-level summary counts in natural Chinese.

## Technical Evidence

Capture detailed chord cards, note-level filters, evidence, warnings, and validation hints.

Suggested caption:

> Technical Evidence keeps deterministic analysis auditable through structured chord evidence, note filters, and backend warnings.

## Download Learning Report

Capture the Generate Learning Report button, the generated Markdown report, and the Download button.

Suggested caption:

> Learning Report exports the current deterministic analysis as Markdown. Users can copy or download it as a `.md` file.

## Screenshot Checklist

- Use a valid `.musicxml` sample.
- Confirm backend and frontend are running locally.
- Capture the Score Input Workspace as the first screenshot.
- Capture both Student Analysis and Technical Evidence.
- Include at least one screenshot showing the score preview and one showing the Learning Report with download.
- Do not imply PDF/image/OMR or real LLM support in captions.
